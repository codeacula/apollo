# Summary

Apollo should support being invited into Discord servers that are managed by other people, not just direct/private usage. This is an important step toward broader real-world testing and making Apollo useful in group environments.\n\n## Goals\n\n- allow a server owner/admin to invite Apollo into their server\n- define what server-level behavior Apollo should support once installed\n- make installation reliable and understandable for non-developer users\n\n## Scope\n\n- installation flow and entry points\n- server vs DM behavior expectations\n- server-level configuration and default behavior\n- audit what assumptions in the current bot flow only work for DMs\n\n## Acceptance Criteria\n\n- a clear install flow exists for server owners/admins\n- Apollo can distinguish DM behavior from server behavior cleanly\n- server-install requirements and limitations are documented\n- there is a path for server-scoped configuration/management over time\n

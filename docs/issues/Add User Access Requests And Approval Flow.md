# Summary

Apollo should provide a clearer way for users to request access instead of failing silently or requiring manual out-of-band coordination.\n\n## Goals\n\n- let users request access when they do not currently have it\n- provide a path for review/approval by an admin or owner\n- make the user experience understandable when access is denied or pending\n\n## Scope\n\n- request-access user flow\n- pending/requested access state\n- admin review/approval workflow\n- messaging/notifications around approval status\n\n## Acceptance Criteria\n\n- a user can request access through Apollo-supported surfaces\n- Apollo can distinguish denied vs pending vs granted access\n- an admin has a defined way to review and approve/reject requests\n- the status is visible enough that users know what is happening\n

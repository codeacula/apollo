# Summary

Extend reminders from single-send behavior into repeatable and manageable product flows.

## Goals

- support recurring reminders for long-running or repeated tasks
- let users list, reschedule, snooze, and cancel reminders directly
- improve planning context by adding due dates to to-dos

## Included Issues

- `docs/issues/Add Recurring Reminders.md`
- `docs/issues/Add Basic Reminder Management Flows.md`
- `docs/issues/Add The Ability To Set Due Date In To-Dos.md`

## Exit Criteria

- recurring reminders can reschedule themselves reliably after sends
- users have a basic but dependable set of reminder management actions
- to-dos can carry due-date context that works with reminder planning

## Notes

- This milestone should follow the core follow-up infrastructure from Milestone 2 so reminder state handling stays coherent
